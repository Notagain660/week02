## 如何优化：后端解析 JSON 并返回对象（结构化）

### 步骤 1：创建统计结果 VO

java

@Data
@AllArgsConstructor
@NoArgsConstructor
public class StatisticsVO {
    private List<Map<String, Object>> items;  // 或 List<ItemCount>
    private String summary;
}

### 步骤 2：修改 `AdminService.statistics()` 解析 AI 返回的字符串

java

public StatisticsVO statistics() {
    List<Map<String, Object>> rawItems = statis.selectItems();
    String aiOutput = aiService.generateStatistics(rawItems);
    // 提取 JSON 部分（修复原来的错误写法）
    int start = aiOutput.indexOf("{");
    int end = aiOutput.lastIndexOf("}") + 1;
    if (start == -1 || end == 0) {
        throw new BusiException(StatusCode.COLLAPSED, "AI 返回格式错误");
    }
    String jsonPart = aiOutput.substring(start, end);
    // 解析 JSON
    ObjectMapper mapper = new ObjectMapper();
    try {
        JsonNode root = mapper.readTree(jsonPart);
        ArrayNode itemsNode = (ArrayNode) root.get("items");
        String summary = root.get("summary").asText();
        // 将 itemsNode 转换为 List<Map>
        List<Map<String, Object>> items = mapper.convertValue(itemsNode, new TypeReference<>() {});
        return new StatisticsVO(items, summary);
    } catch (JsonProcessingException e) {
        throw new BusiException(StatusCode.COLLAPSED, "AI 解析失败");
    }
}

### 步骤 3：修改 Controller 返回类型

java

@GetMapping("/admin/statistics/item")
public MapperResult<StatisticsVO> item() {
    return MapperResult.success(StatusCode.OK, adminService.statistics());
}

这样前端直接取 `response.data.items` 和 `response.data.summary` 即可，无需二次解析。
