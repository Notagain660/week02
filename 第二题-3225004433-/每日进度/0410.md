##### mybatis-plus字段插入
MyBatis-Plus 中 `@TableField` 有一个 `insertStrategy` 属性（以及 `updateStrategy`），用于控制**当实体类字段为 `null` 时，是否要插入或更新该字段**。
默认策略是 `FieldStrategy.DEFAULT`（通常等同于 `NOT_NULL`），即：如果字段为 `null`，则生成的 SQL 中**不包含该列**，数据库会使用列的默认值（如果有）或保留原值。
**可选值**：
- `FieldStrategy.NOT_NULL`：字段非 `null` 时才插入/更新
- `FieldStrategy.NOT_EMPTY`：字段非 `null` 且非空字符串时才插入/更新（适用于字符串）    
- `FieldStrategy.IGNORED`：无论是否为 `null` 都插入/更新（可能覆盖数据库默认值）    
- `FieldStrategy.NEVER`：永远不插入/更新该字段

------以下是swaggerUIauthorize的配置分析（使页面出现authorize按钮并且能传递token）-----

## 一、背景：为什么需要这个配置？

Swagger UI 默认不知道你的 API 使用了什么样的认证方式（比如 JWT Bearer Token）。如果你不告诉它，它就不会显示“Authorize”按钮。这个配置就是在告诉 Swagger UI：**“我的 API 需要 Bearer Token 认证，请提供一个按钮让用户输入 token”**。

---

## 二、逐行解释

### 1. 导入的类（import）

java

import io.swagger.v3.oas.models.Components;
import io.swagger.v3.oas.models.OpenAPI;
import io.swagger.v3.oas.models.info.Info;
import io.swagger.v3.oas.models.security.SecurityRequirement;
import io.swagger.v3.oas.models.security.SecurityScheme;

这些是 **SpringDoc OpenAPI 3** 的模型类，用于构建 OpenAPI 规范文档（即 `/v3/api-docs` 返回的 JSON）。
- `OpenAPI`：整个 API 文档的根对象。
- `Info`：API 的元信息（标题、版本、描述）。
- `Components`：可重用的对象，比如安全方案、参数、响应等。
- `SecurityRequirement`：声明 API 需要哪些安全方案（比如 Bearer Token）。
- `SecurityScheme`：定义一个安全方案（比如 HTTP Bearer Token 的细节）。
### 2. 配置类与 Bean 方法

java

@Configuration
public class SwaggerConfig {
    @Bean
    public OpenAPI customOpenAPI() {
        // 返回一个 OpenAPI 对象
    }
}
- `@Configuration`：表明这是一个 Spring 配置类，会被 Spring 扫描。
- `@Bean`：方法返回的 `OpenAPI` 对象会被 Spring 管理，并自动合并到 SpringDoc 生成的 API 文档中。
### 3. 构建 `OpenAPI` 对象

java

return new OpenAPI()
        .info(new Info()
                .title("校园失物招领平台 API")
                .version("1.0")
                .description("需要认证的接口请在请求头中添加 Authorization: Bearer <token>"))
        .addSecurityItem(new SecurityRequirement().addList("BearerAuth"))
        .components(new Components()
                .addSecuritySchemes("BearerAuth",
                        new SecurityScheme()
                                .name("BearerAuth")
                                .type(SecurityScheme.Type.HTTP)
                                .scheme("bearer")
                                .bearerFormat("JWT")));

#### (1) `.info(...)` —— API 基本信息

- 设置文档标题、版本、描述。这些信息会显示在 Swagger UI 页面的顶部。
    

#### (2) `.addSecurityItem(new SecurityRequirement().addList("BearerAuth"))`

- `SecurityRequirement`：表示“这个 API 需要什么安全凭证”。
    
- `addList("BearerAuth")`：引用下面定义的一个安全方案，名字叫 `"BearerAuth"`。  
    这会让 Swagger UI 在全局范围内（所有接口）都要求认证，除非某个接口用 `@Operation` 单独覆盖。
    

#### (3) `.components(new Components().addSecuritySchemes("BearerAuth", ...))`

- `Components`：容器，里面可以放安全方案、请求体、参数等。
    
- `addSecuritySchemes("BearerAuth", new SecurityScheme()...)`：  
    注册一个名为 `"BearerAuth"` 的安全方案，具体细节由 `SecurityScheme` 定义。
    

#### (4) `new SecurityScheme()` —— 定义 Bearer Token 的细节

- `.name("BearerAuth")`：这个安全方案的内部名字（与 `addList` 中的名字对应）。
    
- `.type(SecurityScheme.Type.HTTP)`：认证类型是 HTTP 认证。
    
- `.scheme("bearer")`：HTTP 认证的子类型是 Bearer Token（即 `Authorization: Bearer <token>`）。
    
- `.bearerFormat("JWT")`：提示 token 的格式是 JWT（可选，仅用于文档说明）。
---

## 三、这个配置如何解决“没有 Authorize 按钮”？

- 没有这个配置时，SpringDoc 不知道你的 API 有认证需求，所以不显示 Authorize 按钮。
    
- 有了这个配置，SpringDoc 生成的 OpenAPI JSON 中会包含安全方案定义，Swagger UI 读取后就会在右上角显示 **“Authorize”** 按钮。
    
- 用户点击后输入 `Bearer <token>`，Swagger UI 会在后续所有请求中自动添加 `Authorization` 头。
---

## 四、为什么你之前“配 Swagger 依赖失败”？

可能的原因：

- 没有添加 `springdoc-openapi-starter-webmvc-ui` 依赖（你后来加了，版本 2.8.16）。
    
- 依赖版本与 Spring Boot 版本不兼容（你当前是 3.5.13，2.8.16 兼容）。
    
- 没有写这个配置类，所以默认安全方案为空。
    
- 或者配置类没有被 Spring 扫描到（检查包路径，确保在主启动类所在包或其子包下）。

## 五、补充说明

- 如果你只想让**部分接口**需要认证，可以在 Controller 的方法上使用 `@Operation(security = @SecurityRequirement(name = "BearerAuth"))`，并移除全局的 `.addSecurityItem`。
    
- 也可以给不同的接口组配置不同的安全要求，但当前项目用全局足够。
现在你把这个配置类加到项目中，重启后 Swagger UI 就会出现 Authorize 按钮了。
