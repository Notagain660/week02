#### ollama配置
1. 下载：
2. 安装（全部默认，不要通过剪切粘贴的方式移动文件）
3. 拉取模型：ollama pull model
>验证模型可用：`ollama run qwen:0.5b --prompt "Hello"`或者验证调用（新的cmd窗口）：`curl http://localhost:11435/api/generate -d "{\"model\":\"qwen:0.5b\",\"prompt\":\"Hi\",\"stream\":false}"`

>这样就成功了：`{"model":"qwen:0.5b","created_at":"2026-04-09T03:06:45.5908247Z","response":"Hi! How can I help you today?","done":true,"done_reason":"stop","context":[151644,872,198,9707,151645,198,151644,77091,198,13048,0,2585,646,358,1492,498,3351,30],"total_duration":1772699200,"load_duration":1584973800,"prompt_eval_count":9,"prompt_eval_duration":66906000,"eval_count":10,"eval_duration":102176500}`

4. 启动服务：`ollama serve`（默认监听 `http://localhost:11434`
>端口占用：
>     `Error: listen tcp 127.0.0.1:11434: bind: Only one usage of each socket address (protocol/network address/port) is normally permitted.`
>     查看端口进程：netstat -ano | findstr :11434
>     强制停止进程：taskkill /PID 进程PID（LISTENING后面的数字） /F
>     验证服务是否启动：另开CMD，
     `curl http://localhost:11434/api/generate -d "{\"model\":\"qwen:0.5b\",\"prompt\":\"hi\",\"stream\":false}"`
     
>端口被反复占用（删掉一个出现一个）：换端口，系统环境变量OLLAMA_HOST，地址127.0.0.1:pid，配置文件和使用中的url记得改

>检查服务状态：ollama list

查看端口占用：`netstat -ano | findstr :pid`
杀进程:`taskkill /PID pid /F`
    子进程也终止:`taskkill /PID pid /F /T`
停止ollama服务:`net stop ollama`
检查是否有ollama服务:`sc query ollama`
禁用ollama服务:`sc config ollama start= disabled`
`net stop winnat`
`netsh interface ipv4 add excludedportrange protocol=tcp startport=11434 numberofports=1`
`net start winnat`
**操作详解：**
- **`net stop winnat`**：临时停止 Windows NAT 服务，为修改端口配置做准备。
- **`netsh interface ipv4 add excludedportrange ...`**：这是核心命令，明确告诉系统，`11434` 这个端口是“保留”的，不要自动分配给其他程序。这样，它就不会再被系统服务强行占用了。
- **`net start winnat`**：重新启动 NAT 服务，让配置生效。



5. 在 Spring Boot 中调用 Ollama API
    使用 `RestClient`（你已引入 `spring-boot-starter-restclient`）
    **配置类**（读取配置，创建 RestClient）例如：
    @Configuration
    public class OllamaConfig {
        @Bean
        public RestClient ollamaRestClient() {
            return RestClient.builder()
                .baseUrl("http://localhost:pid")
                .defaultHeader("Content-Type", "application/json")
                .build();
     }
    }
