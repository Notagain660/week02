- 聊天（非实时通信）：
    - 发消息：校验关系和用户状态之后发即可
    - 删除/已读消息：均为修改消息状态
    - 查看消息：因为只筛掉已删除的消息所以用stream流操作
        -  `return chats.stream()`
            - `chats.stream()`：将 `List<Chat>` 转换为一个 Java 8 的 **Stream**（流），以便进行函数式操作（如过滤、排序、收集）。
        - `.filter(chat -> { ... })`
            - `filter` 是 Stream 的中间操作，接受一个 **Predicate**（判断条件），返回值是 `boolean`。只有返回 `true` 的元素会保留，`false` 的被过滤掉。
            - `chat -> { ... }` 是 **Lambda 表达式**，参数 `chat` 代表当前遍历到的每一条聊天记录（`Chat` 对象）。大括号内是多行代码块，最后需要返回 `boolean`。
        - `if (chat.getSenderId() == currentUserId) {`
            - 判断：这条消息的**发送方ID**是否等于当前登录用户的ID。
            - 如果成立，说明**当前用户是这条消息的发送者**
        - `return chat.getChatStatus() != ChatStatus.MEDELETED;`
            - 如果当前用户是发送者，则检查这条消息的状态**不是** `MEDELETED`（发送方已删除）。
            - `!=` 表示“不等于”。如果状态不是已删除，则保留；如果是已删除，则过滤掉。
            - 注意：`MEDELETED` 是你定义的枚举，表示“发送方删除了这条消息”。
        -  `} else {`
            - 否则（当前用户不是发送者，即当前用户是接收者）。
        - `return chat.getChatStatus() != ChatStatus.OPDELETED;`
            - 如果当前用户是接收者，则检查消息状态**不是** `OPDELETED`（接收方已删除）。
            - 如果状态不是接收方删除，则保留；如果是，则过滤掉。
        - `.sorted(Comparator.comparing(Chat::getSendTime))`
            - `sorted` 是 Stream 的中间操作，用于对流中的元素进行排序。
            - `Comparator.comparing(Chat::getSendTime)`：创建一个比较器，按照 `Chat` 对象的 `getSendTime()` 方法返回的 `LocalDateTime`（发送时间）进行**升序**排序（最早的在前）
        - `.collect(Collectors.toList())`
            - `collect` 是 Stream 的终止操作，将处理后的流重新收集为一个 `List<Chat>`。
            - `Collectors.toList()` 是 JDK 内置的工具，用来生成一个新的列表。
