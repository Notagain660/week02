##### 统计：
- **思路**：
    - 第一版：先统计全部物品的原始名称频次，再让 AI 合并语义相近的项，最后只输出频率最高的前十项。
    - 第二版：直接让ai根据全部统计结果生成一句话总结，不用解析，生成图表什么的把统计结果给前端来做就行

- 活跃用户：近7/30天发布过内容（post/comment），示例：
````{SQL：
    SELECT posterid AS userid FROM post WHERE posttime BETWEEN #{start} AND #{end} +
    UNION  +
    SELECT commenterid FROM comment WHERE replytime BETWEEN #{start} AND #{end}
    
在mapper类：List<Long> selectActiveUserIds(@Param("start") LocalDateTime start, @Param("end") LocalDateTime end);
}
````
- 时间输入： `startDate.atStartOfDay()` 是 `LocalDate` 的方法，用来得到 `2025-04-01T00:00:00`。
    - 用 `endDate.plusDays(1).atStartOfDay()` 得到次日零点，SQL 中用 `BETWEEN start AND end` 时，`end` 是次日零点，这样实际上包含的是 `[start, end)`，即结束日期当天所有数据。但为了精确，你也可以改用 `>= start AND < end`，但 `BETWEEN` 配合次日零点也等价（只要数据库的 `posttime` 不包含次日零点）。
- `${start}` 是字符串直接替换，存在 SQL 注入风险，且对于 `LocalDateTime` 类型无法正确转换为 SQL 时间格式。应使用 `#{start}` 预编译占位符。

发布信息数：总共有效的，不包括已删除的，示例：
````{SQL：
SELECT COUNT(*) FROM post WHERE poststatus = 0;  -- 0 为未处理/正常状态
service里面使用：long totalPosts = postMapper.selectCount(new LambdaQueryWrapper<Post>().eq(Post::getPostStatus, PostStatus.UNFINISHED));}
````

找回数：lost并且已完成，示例：
````{SQL:
SELECT COUNT(*) FROM post WHERE type = 0 AND poststatus = 3;  -- type=0 丢失, poststatus=3 已完成}
````

- AI总结哪里/什么东西丢的多
    1. 从数据库查询按原始物品名称分组统计的结果（例如 `itemname` 和 `COUNT(*)`）。示例：
        ````{SQL:
        SELECT itemname, COUNT(*)AS cnt 
        FROM post 
        WHERE type = 0   -- 0 表示丢失物品
        GROUP BY itemname
        ORDER BY cnt DESC;
         service里面： List<Map<String, Object>> selectItemStatistics();
        }
        ````
    2. 将统计结果（如 `{"校园卡":20, "伞":12, "雨伞":8, "书包":5, "笔记本":4}`）构造成一个结构化的文本（例如 JSON 或 Markdown 表格）。示例：
        ````{ import cn.hutool.json.JSONUtil;
         List<Map<String, Object>> stats = postMapper.selectItemStatistics();
         String jsonStats = JSONUtil.toJsonStr(stats);}
        ````
    3. 调用 AI，要求它识别出语义相近的物品名称，并合并它们的计数，最后生成一段总结。（prompt）示例：
        ````{
        String prompt = String.format("""
        以下是校园失物招领平台中丢失物品的原始统计（物品名称: 丢失次数）：
        %s
        请完成以下任务：
        1. 将语义相近的物品名称合并为一个通用类别（例如“伞”和“雨伞”合并为“雨伞”）。
        2. 合并后，按丢失次数从高到低排序，输出丢失次数最多的前10个物品类别。
        3. 返回格式必须为 JSON，结构如下：
        {
          "items": [
            {"name": "校园卡", "count": 20},
            {"name": "雨伞", "count": 20},
            ...
          ],
          "summary": "一句话总结丢失热点"
        }只返回 JSON，不要有其他内容。""", jsonStats);
        }
        ````
    4. 解析 AI 返回的内容（通常为 JSON 格式），得到合并后的物品类别及次数，以及总结文本。示例：
        ````{
        ChatResponse response = chatModel.call(new Prompt(prompt));
        String aiOutput = response.getResult().getOutput().getText();
        // 提取 JSON 部分（如果 AI 加了多余说明）（根据ai返回的内容可选）
        int start = aiOutput.indexOf("{");
        int end = aiOutput.lastIndexOf("}") + 1;
        String jsonPart = aiOutput.substring(start, end);
        }
        ````
